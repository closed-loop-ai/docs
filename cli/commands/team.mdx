## 👥 Overview

The `cl team` command helps you manage your team's configuration and settings. The most important setting is your product website URL, which helps ClosedLoop AI understand your product context for more accurate feedback analysis.

## 🚀 Basic Usage

### View Current Website

```bash
# Show current team website
cl team website

# JSON output
cl team website --json
```

### Update Team Website

```bash
# Set your product website
cl team website "https://yourproduct.com"

# JSON output
cl team website "https://yourproduct.com" --json
```

## 📋 Parameters

<ParamField path="website" type="string">
  Your product's website URL. Must include protocol (http:// or https://) and domain.
</ParamField>

<ParamField path="--json" type="flag">
  Output results in JSON format for scripting and automation.
</ParamField>

## 🎯 Why Set Your Product Website?

<Warning>
**This is important!** ClosedLoop AI uses your product website URL to understand your product context, which enables more accurate feedback analysis and insights.
</Warning>

### How It Helps AI Analysis

When you set your product website, ClosedLoop AI can:

- **Learn about your product features** from your website content
- **Understand your competitors** and market positioning
- **Analyze pricing and value propositions**
- **Provide product-specific insights** instead of generic feedback
- **Better categorize feedback** by feature area

## 🔍 Advanced Usage

### Website Validation

The CLI automatically validates website URLs:

```bash
# Valid URLs
cl team website "https://yourproduct.com"        # ✅ Valid
cl team website "http://yourproduct.com"         # ✅ Valid
cl team website "https://yourproduct.com/features" # ✅ Valid

# Invalid URLs
cl team website "yourproduct.com"                # ❌ Missing protocol
cl team website "ftp://yourproduct.com"          # ❌ Invalid protocol
cl team website "not-a-url"                      # ❌ Invalid format
```

### Configuration Management

```bash
#!/bin/bash
# update-website.sh

NEW_WEBSITE="$1"

if [ -z "$NEW_WEBSITE" ]; then
  echo "Usage: $0 <website-url>"
  exit 1
fi

# Update website
cl team website "$NEW_WEBSITE"

# Verify update
CURRENT_WEBSITE=$(cl team website --json | jq -r '.data.website')
echo "Website updated to: $CURRENT_WEBSITE"
```

### Environment-Specific Configuration

```bash
#!/bin/bash
# setup-environment.sh

ENVIRONMENT="$1"

case $ENVIRONMENT in
  "development")
    cl team website "https://dev.yourproduct.com"
    ;;
  "staging")
    cl team website "https://staging.yourproduct.com"
    ;;
  "production")
    cl team website "https://yourproduct.com"
    ;;
  *)
    echo "Usage: $0 <development|staging|production>"
    exit 1
    ;;
esac

echo "Website configured for $ENVIRONMENT environment"
```

## ❌ Error Handling

### Common Errors

<Error>
**Invalid Website URL**
```
❌ Validation Error: Invalid website URL format. Must include protocol (http:// or https://) and domain
```
**Solution**: Ensure URL includes protocol and valid domain.
</Error>

<Error>
**API Key Missing**
```
🔑 No API key configured!
```
**Solution**: Run `cl config set --api-key <your-key>`.
</Error>

<Error>
**Invalid API Key**
```
❌ Invalid API key. Get your free API key at http://app.closedloop.sh
```
**Solution**: Get a new API key from your dashboard.
</Error>

### Error Codes

| Code | Description | Solution |
|------|-------------|----------|
| `VALIDATION_ERROR` | Invalid URL format | Include protocol (http:// or https://) |
| `NO_API_KEY` | No API key configured | Set API key with `cl config set --api-key <key>` |
| `INVALID_API_KEY` | API key is invalid | Get new key from http://app.closedloop.sh |
| `API_ERROR` | Server error | Check internet connection, try again |

## 🎯 Best Practices

### 1. Use Specific URLs

```bash
# Good - specific feature page
cl team website "https://yourproduct.com/features/analytics"

# Also good - main product page
cl team website "https://yourproduct.com"

# Bad - too generic
cl team website "https://company.com"
```

### 2. Update for Different Environments

```bash
# Good - environment-specific URLs
cl team website "https://yourproduct.com"           # Production
cl team website "https://staging.yourproduct.com"   # Staging
cl team website "https://dev.yourproduct.com"       # Development
```

### 3. Use JSON for Automation

```bash
# Good - structured output
WEBSITE=$(cl team website --json | jq -r '.data.website')

# Bad - parsing text output
WEBSITE=$(cl team website | grep "Website:" | cut -d' ' -f2)
```

### 4. Validate Before Setting

```bash
#!/bin/bash
# validate-and-set.sh

WEBSITE="$1"

# Basic validation
if [[ ! "$WEBSITE" =~ ^https?:// ]]; then
  echo "Error: URL must include protocol (http:// or https://)"
  exit 1
fi

# Set website
cl team website "$WEBSITE"

# Verify
CURRENT=$(cl team website --json | jq -r '.data.website')
echo "Website set to: $CURRENT"
```
